Backend Workflows

Key data and process flows connecting the four surfaces.

Document Ingestion Pipeline

Customer submits receipt via WhatsApp, email, or upload → receipt gateway ingests into Coding Engine → ML extraction (category, amount, supplier) → accountant reviews in document queue → coded document published to SleekBooks ledger as journal entry or invoice.

Bank Reconciliation Flow

Bank statement imported via customer portal or SleekBooks → transactions extracted → ML auto-reconciliation matches transactions to existing documents → unmatched items surfaced in Coding Engine for manual review → confirmed matches synced back to SleekBooks and marked reconciled.

Client Onboarding & KYC

Client signs up → Admin App triggers onboarding Camunda workflow → KYC verification via Onfido → company record created in SleekBooks (ERPNext) → accountant assigned in Coding Engine → customer portal accounting section unlocked.

Ledger Sync Pipeline

Coding Engine publishes a coded document → webhook fires to sleek-erpnext-service → journal entry or invoice created in ERPNext → financial reports (GL, BS, P&L) updated in SleekBooks → optionally synced to Xero via xero-sleekbooks-service for clients using Xero as their source of truth.
