## Evidence

### sleek-back — `@sleek-sdk/sleek-cms`

- `package.json` — dependency `@sleek-sdk/sleek-cms`.
- `utils/app-features-util.js` — `AppFeatureService`, `AppFeature`, `cms-settings`, optional `config/app-features.json` override; `sync-app-features` script in `package.json` for local feature sync.
- `controllers-v2/handlers/config/platform-config.js` — builds `cmsAppFeatures`, `cmsGeneralFeatureList`, merges partner/tenant feature payloads for `/v2/config/platform/:platform`.
- `controllers-v2/config.js` — routes `/v2/config/platform/customer`, `/admin`, `/partner`, `/platform/authenticated`, `/sleek-address`, dynamic `/:platform`.
- `vendors/mailer-vendor.js` — `EmailTemplateService` from `@sleek-sdk/sleek-cms` for CMS-backed mail templates.

### sleek-back — runtime sync and cache

- `services/cms-event-notification-service.js` — Kafka consumer (`cms-cors-whitelisted-url-sleekback-notification`), CORS whitelist list, `emitRefreshCorsWhitelistedDomain`, model constants for CMS entry events.
- `controllers/cms-controller.js` — `POST /cms/cache/update`: `CacheManager.resetCache()` (`@sleek-sdk/sleek-cms`), `CmsEventNotificationService.emitRefreshCorsWhitelistedDomain`, receipt feature cache refresh.
- `app.js` — `CmsEventNotificationService.getInstance()`, registration with whitelist used by `/api/health`.

### customer-mfe

- `platform-config-store.js` (e.g. onboarding, dashboard) — `GET /v2/config/platform/customer/`.
- Components using `platformConfig.cmsAppFeatures`, `cmsGeneralFeatureList` (e.g. onboarding, Auth0-related props).

### sleek-website

- `src/utils/api-utils.js`, admin views — `platformConfig.cmsAppFeatures`, `cmsGeneralFeatureList` (e.g. `page-header/layout.js`, `view.js`).
