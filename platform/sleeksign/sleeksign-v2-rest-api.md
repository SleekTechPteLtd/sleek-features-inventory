## Evidence

### sleek-sign-backend

- `app.js` — `app.use("/v2", tenantIdGateway, require("./routes-v2"))`.
- `routes-v2/index.js` — route list: document drafts, WFE free-user upload/upgrade, OTP, social-auth-init, document delete/reassign, send-reminders, admin void/dynamic-fields/mark-as-complete, contacts, templates, free-user check-email/create-account, shared `errorHandler`.
- `routes-v2/documents.js` — handlers for admin and document mutations (imports platform third-party proxy, stamper SQS, etc.).

### customer-mfe

- `sleek-sign/src/proxies/back-end/sleeksign/`* — many calls target sign backend or **sleek-back** depending on proxy; cross-check each proxy’s `backendUrl` when tracing.
